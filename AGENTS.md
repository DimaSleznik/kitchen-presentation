# AGENTS.md — Kitchen Planner · презентационная борда

> Контекст и правила для любого агента/разработчика, кто трогает этот проект.
> Главное прочитать целиком до первой правки: тут зафиксировано **зачем** проект и
> **для кого** — от этого зависит каждое решение.

## Зачем этот проект существует

`kitchen-presentation` — самодостаточный статический сайт-презентация. Он **упаковывает**
технический проект «параметрический каталог кухонь» так, чтобы его понял **нетехнический
человек**. Сайт деплоится на **GitHub Pages** и отдаётся коллеге-переговорщику: он открывает
его сам, без нас, и должен за 10–15 минут разобраться:

- зачем этот проект и какую проблему решает;
- как устроена архитектура (вкладка «Как устроено» / стенды);
- какой роадмап и эстимейты (вкладка «План работ» / борда);
- что показывать заказчику на каждом этапе.

Это **не** продакшен-приложение и **не** часть прототипа. Это витрина: её работа — объяснить
и убедить. Если что-то на сайте не помогает нетехнарю понять или поверить — оно лишнее.

## Аудитория определяет всё

Один читатель: **переговорщик / ЛПР, не инженер**. Ведёт переговоры с заказчиком (бренд
**Häcker**). Открывает сайт холодным, один, без нас рядом. Отсюда правила:

- **Ведём за руку.** Каждый экран сначала говорит «что это и зачем», потом детали.
- **Бизнес-язык, не жаргон.** Никаких `Zustand`, `orchestrator.tsx 1849 строк`, `InstancedMesh`,
  `tool-calling`. Вместо этого — «состояние сцены», «слишком большой файл = риск по срокам,
  лечим», «повторяем мебель дёшево», «ассистент отвечает строго по правилам».
- **Картинка > абзац.** Ключевые идеи — через плашки-объяснялки (см. ниже), не стеной текста.
- **Понятные тайтлы, тултипы, объяснялки.** Высокий UX: минимум усилий, чтобы понять.

> **Гард-рейл.** Если в тексте, который видит пользователь, появляется имя файла/функции/
> библиотеки или аббревиатура без расшифровки — это **баг**. Глубокая техника живёт в
> свёрнутом разделе «для технарей», а не в основном потоке.

## Стек сайта

- **Vite 6**, ванильный JS (ESM), без фреймворка. Hash-роутер вручную в `src/main.js`.
- Контент: markdown (`marked`) + `mermaid` + `highlight.js`. Шрифты self-hosted
  (`@fontsource-variable`) — работает офлайн.
- Деплой: статика из `dist/` → GitHub Pages.

## Структура

| Файл | Роль |
|---|---|
| `index.html` | Каркас + «Обзор» (лендинг, ведёт за руку) |
| `src/main.js` | Роутер, рендер доков, reveal-анимации |
| `src/board.js` | Вкладка «План работ»: фазы × роли, карточки, стоперы, панель-объяснялка |
| `src/arch.js` | Вкладка «Как устроено»: зоны → блоки → связи |
| `src/plates.js` + `src/plates/*.svg` | Система плашек-объяснялок (переиспользуемые SVG) |
| `src/style.css` | Единая палитра/токены, все стили |
| `docs/*.md` | Документы (бизнес-язык + свёрнутая техника) |

Контент **data-driven**: правь массивы (`LANES`, `ITERATIONS`, `TASKS`, `NODES`, `EDGES`),
а не разметку.

## Стандарт плашек-объяснялок

Взят из Knight Fantasy (`Docs/05 🔧 Инструкции/Иллюстрации-плашки для объяснялок.md`),
адаптирован под нашу палитру. Самодостаточные SVG-карточки:

- одна понятная идея на картинку, через метафору;
- role-цвета нашей палитры + однострочная легенда, если цвет кодирует роль;
- **текст не вылезает** за блок (длинный — под `textLength`+`lengthAdjust`);
- нет пустых блоков; подписи на стрелках — с halo-подложкой, не пересекают линию;
- `role="img"` + `<title>` + `<desc>` (одно предложение по-русски);
- цифры/детали — в прозе под плашкой, не внутри блоков.

> **Обязательная проверка:** отрастеризовать SVG в PNG (headless Chrome) и **прочитать
> картинку глазами** — XML-валидности мало (текст не переносится, поздний `<rect>` рисует
> поверх подписи, стрелка может пройти сквозь блок). Рецепт — в спеке редизайна
> `docs/superpowers/specs/2026-06-24-board-redesign-design.md`.

## Принципы при правках

- Любой новый текст для пользователя проходит гард-рейл «без жаргона».
- Ключевую идею показывай **плашкой**, а не абзацем.
- Гранулярность борды — **верхнеуровневая**: одна понятная задача на роль в фазе; детали — в `note`.
- Не ломай data-driven подход и единую палитру.
- Сначала «что это и зачем», потом «как именно».
