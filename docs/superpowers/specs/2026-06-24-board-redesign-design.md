# Спека · Редизайн борды «ведём за руку» для нетехнического переговорщика

Дата: 2026-06-24 · Статус: утверждён, в работе

## Контекст и цель

`kitchen-presentation` — статический сайт под GitHub Pages, который упаковывает технический
проект «параметрический каталог кухонь» для **нетехнического переговорщика** (ведёт переговоры
с Häcker, открывает сайт сам). Прототип, который оценивается, — `C:\UnityProject\Kitchen`.
Подробно про «зачем» и аудиторию — в `AGENTS.md`.

Цель редизайна: поднять UX «в разы» — вести за руку, бизнес-язык, сильные объяснялки, крутые
кастомные SVG (стандарт «плашек» из Knight Fantasy). Вкладки «План работ» и «Как устроено»
остаются отдельными.

## Подход

Эволюция (не ребилд): движки `board.js`/`arch.js` остаются, меняется костяк — навигация,
система плашек, тексты на бизнес-языке, переупаковка техники.

## Дизайн по разделам

### 1. Навигация и «вход»
Было 8 технических пунктов. Стало 4 понятных:
- **Обзор** (`#/`) — лендинг, ведёт за руку: что это, проблема→решение, как смотреть, CTA в план.
- **План работ** (`#/board`) — борда (фазы × роли).
- **Как устроено** (`#/arch`) — стенды (зоны → блоки → связи).
- **Документы** (`#/docs`) — индекс доков на бизнес-языке; глубокая техника помечена «для технарей».

Каждая вкладка открывается строкой «что это и зачем» + плашкой «как читать этот экран».

### 2. Система плашек-объяснялок
Новый модуль `src/plates.js` + `src/plates/*.svg`. Плашки — самодостаточные SVG по стандарту
(см. AGENTS.md). `plates.js`: `import.meta.glob('./plates/*.svg', {query:'?raw', eager:true})`,
экспорт `plate(name, caption)` → `<figure class="plate">…<figcaption>`.

Стартовый набор:
- `thesis` — тысячи файлов → десятки шаблонов (главный тезис).
- `read-board` — как читать борду (фазы→, роли↓, ⛔ стопер, ★ что показать).
- `read-arch` — как читать карту (клиент / сервисы / данные / внешние).
- `three-layers` — 3 слоя данных (геометрия / коммерция / правила).
- `hybrid` — гибрид: корпус процедурно + детали-ассеты.
- `ai-roles` — 3 роли ИИ (наполнение / ассистент / превью).
- `maturity` — зрелость и риски (замена «техдолга» на бизнес-язык).

### 3. Борда — объяснялки сильнее
Панель карточки (`drawer`) переписывается по схеме: **хук → зачем простыми словами → что
увидит заказчик → стопер → (мини-плашка где помогает) → ссылка**. Каждой задаче в данных
добавляются поля `hook` (1 строка-зацепка) и `plain` (зачем без жаргона); `note` остаётся
как «детали». Онбординг-плашка `read-board` над сеткой. Тултипы на ховере карточки.

### 4. Стенды — бизнес-язык
Описания блоков `NODES[].desc` переписать как «что это даёт пользователю/бизнесу». Добавить
поле `plain`. Плашка `read-arch` в интро.

### 5. Техника → бизнес-язык
Вкладка «Техдолг» убирается. Вместо неё раздел/док **«Зрелость и риски»**: не имена файлов,
а «насколько код готов к росту, где риски по срокам, как снимаем». Плашка `maturity`. Доки
`backend-services`, `catalog-architecture` получают человеческое интро; глубокие части — под
заголовком «Для технарей».

### 6. AGENTS.md
Закрепляет зачем/для кого/связь с прототипом/стандарт плашек. (Готов.)

## Проверка плашек растеризацией (обязательно)
```bash
# headless Chrome рендерит SVG 1:1 по viewBox; затем Read по PNG
CHROME="/c/Program Files/Google/Chrome/Application/chrome.exe"
SVG="src/plates/thesis.svg"; PNG="$TEMP/plate.png"
"$CHROME" --headless=new --disable-gpu --hide-scrollbars --force-device-scale-factor=2 \
  --default-background-color=ffffffff --window-size=780,430 --screenshot="$PNG" "file:///$PWD/$SVG"
# Read $PNG глазами: overflow / наезды / сквозные стрелки / пустота / баланс
```

## План реализации (порядок)
1. `src/plates.js` + первый плашка-файл `thesis.svg` (паттерн зафиксирован сэмплом). Проверить PNG.
2. Остальные плашки набора (`read-board`, `read-arch`, `three-layers`, `hybrid`, `ai-roles`, `maturity`). Проверить PNG.
3. `index.html` + `src/main.js`: новая навигация (4 пункта), маршрут `#/board`, `#/docs`, «Обзор» как `#/`.
4. «Обзор» (лендинг) — переписать `<main>`: хук, проблема→решение, плашки, CTA.
5. `board.js` — поля `hook`/`plain`, новый drawer, онбординг-плашка, тултипы, бизнес-тексты.
6. `arch.js` — `plain`-описания, плашка read-arch.
7. Техдолг → «Зрелость и риски»: убрать вкладку, добавить раздел/док + плашка `maturity`.
8. «Документы» — индекс на бизнес-языке, пометки «для технарей».
9. `npm run build` зелёный + визуальная проверка ключевых экранов.
